\section{Wstęp teoretyczny}

Projekt dotyczy implementacji i analizy efektywności algorytmu Symulowanego Wyżarzania (ang. Simulated Annealing, SA) 
dla asymetrycznego problemu komiwojażera (ATSP). Algorytm SA należy do klasy metaheurystyk inspirowanych procesami fizycznymi
- w tym przypadku procesem wyżarzania metali, w którym materiał jest podgrzewany do wysokiej temperatury, a następnie powoli schładzany,
co pozwala atomom na osiągnięcie stanu o niskiej energii.

\subsection{Opis problemu}

Asymetryczny problem komiwojażera (ATSP) polega na znalezieniu najkrótszej trasy odwiedzającej każde miasto dokładnie raz
i wracającej do punktu startowego, przy czym koszt przejścia z miasta $i$ do miasta $j$ może być różny od kosztu przejścia z $j$ do $i$.
Problem ten jest NP-trudny, co oznacza, że nie istnieje znany algorytm wielomianowy gwarantujący znalezienie rozwiązania optymalnego
w rozsądnym czasie dla dużych instancji.

\subsection{Algorytm Symulowanego Wyżarzania}

Symulowane Wyżarzanie jest algorytmem przeszukiwania lokalnego z mechanizmem ucieczki z minimów lokalnych.
W odróżnieniu od klasycznego przeszukiwania lokalnego, SA akceptuje również rozwiązania gorsze od bieżącego
z pewnym prawdopodobieństwem zależnym od temperatury i wielkości pogorszenia.

\subsubsection{Elementy algorytmu}

\begin{itemize}
    \item \textbf{Rozwiązanie początkowe} - punkt startowy przeszukiwania. Może być generowane losowo lub za pomocą heurystyki (np. Nearest Neighbour).
    \item \textbf{Sąsiedztwo} - zbiór rozwiązań osiągalnych z bieżącego rozwiązania przez pojedynczą modyfikację. W implementacji zastosowano zamianę dwóch losowych miast w ścieżce.
    \item \textbf{Temperatura} - parametr kontrolujący prawdopodobieństwo akceptacji gorszych rozwiązań. Wysoka temperatura oznacza większą eksplorację, niska - eksploatację.
    \item \textbf{Schemat chłodzenia} - funkcja opisująca jak temperatura maleje w czasie. Zastosowano schemat geometryczny: $T_{k+1} = \alpha \cdot T_k$, gdzie $\alpha \in (0, 1)$.
    \item \textbf{Kryterium akceptacji} - rozwiązanie lepsze jest zawsze akceptowane. Rozwiązanie gorsze o $\Delta$ jest akceptowane z prawdopodobieństwem $P = e^{-\Delta / T}$.
    \item \textbf{Epoka} - liczba iteracji wykonywanych na jednym poziomie temperatury przed schłodzeniem.
    \item \textbf{Kryterium stopu} - limit czasu działania algorytmu.
\end{itemize}

\subsubsection{Pseudokod}

\begin{verbatim}
Funkcja SimulatedAnnealing(MacierzKosztow, N, Config):
    S = GenerujRozwiazaniePoczatkowe(N, Config.typ_poczatkowy)
    KosztS = ObliczKoszt(S)
    Najlepsze = S
    NajlepszyKoszt = KosztS
    T = Config.temperatura_poczatkowa
    
    Dopóki (T > T_min ORAZ czas < limit):
        Wykonaj Config.dlugosc_epoki razy:
            i, j = LosoweIndeksy(1, N-1)
            Delta = ObliczDeltaSwap(S, i, j)
            
            Jeśli Delta < 0:
                WykonajSwap(S, i, j)
                KosztS += Delta
            W przeciwnym razie:
                P = exp(-Delta / T)
                Jeśli Random(0,1) < P:
                    WykonajSwap(S, i, j)
                    KosztS += Delta
                    
            Jeśli KosztS < NajlepszyKoszt:
                NajlepszyKoszt = KosztS
                Najlepsze = S
                
        T = T * alpha  // chlodzenie geometryczne
        
    Zwróć NajlepszyKoszt, Najlepsze
\end{verbatim}

\subsubsection{Złożoność obliczeniowa}

Złożoność jednej iteracji algorytmu SA wynosi $O(1)$ dzięki obliczaniu delty kosztu zamiast przeliczania całej ścieżki.
Całkowita złożoność zależy od liczby iteracji, która jest kontrolowana przez limit czasu i parametry chłodzenia.
Dla schematu geometrycznego liczba poziomów temperatury wynosi $\frac{\ln(T_{min}/T_0)}{\ln(\alpha)}$.

\textbf{Zalety}:
\begin{itemize}
    \item Zdolność do ucieczki z minimów lokalnych
    \item Prostota implementacji
    \item Dobre wyniki dla wielu problemów kombinatorycznych
    \item Możliwość kontroli jakości rozwiązania przez parametry
\end{itemize}

\textbf{Wady}:
\begin{itemize}
    \item Nie gwarantuje znalezienia rozwiązania optymalnego
    \item Wrażliwość na dobór parametrów (temperatura, chłodzenie, epoka)
    \item Czas działania może być długi dla dużych instancji
\end{itemize}

\subsection{Metodologia}

\begin{itemize}
    \item Każda konfiguracja została uruchomiona 10 razy dla każdej instancji.
    \item Mierzono średni błąd względny, średni czas znalezienia najlepszego rozwiązania oraz najlepszy znaleziony wynik.

\section{Wnioski}

\begin{enumerate}
    \item Algorytm SA dla większości badanych instancji (16 z 18) osiąga błąd poniżej 15\%. Wyjątkiem jest instancja ftv170 (N=171), dla której błąd wyniósł 51.43\% (Random) i 42.40\% (NN) - wynika to z niewystarczającego czasu działania (60s) przy zastosowanych parametrach dla tej konkretnej struktury grafu.
    
    \item Dla małych instancji (N $\leq$ 48) algorytm często znajduje rozwiązanie optymalne lub bliskie optimum (błąd < 5\%). Dotyczy to instancji br17, ftv33, ftv35, ftv38, p43, ry48p.
    
    \item Wpływ rozwiązania początkowego nie jest jednoznaczny:
    \begin{itemize}
        \item Dla ftv170 start z NN daje wyraźnie lepszy wynik (42.40\% vs 51.43\%), ponieważ NN dostarcza lepszy punkt startowy dla dużej instancji.
        \item Dla mniejszych instancji różnice są minimalne - SA z dostatecznym czasem jest w stanie uciec z lokalnego minimum niezależnie od startu.
        \item Dla rbg358 start losowy daje lepszy wynik (14.02\% vs 16.68\%), co sugeruje że NN może prowadzić do gorszego minimum lokalnego.
    \end{itemize}
    
    \item Czas znalezienia najlepszego rozwiązania nie wykazuje prostej korelacji z rozmiarem instancji. Wynika to z losowej natury algorytmu i restartu temperatury - najlepsze rozwiązanie może zostać znalezione w dowolnym momencie działania.
    
    \item Instancje rbg (N=323-443) mimo dużego rozmiaru mają niski błąd (3-14\%), co sugeruje że ich struktura (rzadki graf z małymi wagami) jest korzystna dla operacji swap w SA.
\end{enumerate}
